How to persist logs
Shape a platform_audit_logs table or reuse portal-specific logs
  • Include columns such as school_id, user_id, action, resource_type, resource_id, old_values, new_values, metadata, created_at.
  • For financial operations, you already have accountant_audit_logs which can be surfaced here per school.
  • Attach client information like IP address, user agent and session ID for compliance.
Filters & enforcement
Recommended checks when wiring backend
  • Only Super Admins should query cross-school logs; respect RLS on school-specific tables.
  • Consider a retention strategy (e.g. 12–24 months) and cold storage for older events.
  • Expose download APIs for CSV / JSON exports with server-side filtering and pagination.
